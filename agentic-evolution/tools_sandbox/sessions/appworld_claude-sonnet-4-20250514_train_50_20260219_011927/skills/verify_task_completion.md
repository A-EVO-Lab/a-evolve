## Instructions

Step 1: Review the original task goal and identify key requirements.

Step 2: List the actions you took to complete the task.

Step 3: For each requirement, verify both TECHNICAL and SEMANTIC completion:
- Technical: Does the solution mechanically satisfy the requirement? (e.g., music plays for 85 minutes)
- Semantic: Does the solution match likely user intent? (e.g., distinct songs vs. looping the same songs)

Step 4: Check for semantic mismatches:
- If the task asks for 'enough songs for X minutes', verify you have distinct songs totaling X minutes, not just looping
- If the task asks to 'play music during workout', verify the music variety matches workout expectations
- If the task involves playlists/queues, verify the content provides appropriate variety

Step 5: Compare your solution approach against alternative approaches:
- Would a different approach better match user intent?
- Did you take a shortcut (e.g., looping) when a comprehensive solution (e.g., multiple playlists) was needed?

Step 6: Verify all constraints are satisfied:
- Check numerical constraints (durations, counts, limits)
- Check qualitative constraints (preferences, styles, appropriateness)
- Check implicit constraints (variety, non-repetition, user expectations)

Step 7: If any semantic mismatch or constraint violation is found, revise your solution before claiming completion.

Step 8: Only mark the task as complete when both technical AND semantic requirements are satisfied.
